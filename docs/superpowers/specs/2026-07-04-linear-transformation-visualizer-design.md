## 1. 한 줄 컨셉

> "행렬을 곱하면 도형이 어떻게 움직일까?" — 2×2 행렬로 평면을 변형하며 선형변환이 AI/그래픽스의 기본 언어임을 체험하는 인터랙티브 시각화기.

학생은 행렬 성분을 조절해 도형과 기저벡터가 변형되는 과정을 관찰하고, 이를 **신경망의 가중치 연산·컴퓨터그래픽스 변환**으로 확장 해석한다.

## 2. 목표와 성공 기준

**목표**
- 행렬 단원(행렬의 곱셈, 곱셈의 비교환성)을 2×2 선형변환의 시각으로 이해한다.
- 행렬×벡터 연산과 기저벡터 이동의 관계를 직관적으로 전달한다.
- AI에서 행렬이 왜 핵심인지(신경망 층 = 가중치행렬×입력)를 연결한다.
- GitHub Pages에서 모바일·데스크톱 모두 잘 보이는 결과물을 만든다.

**성공 기준**
- 슬라이더(a,b,c,d) 또는 프리셋으로 변환 행렬을 바꾸면 도형·기저벡터가 실시간으로 변형된다.
- 행렬식(det) = 넓이 배율이 실시간으로 표시된다.
- 회전→전단과 전단→회전의 합성 결과가 다름(곱셈의 비교환성)을 시각적으로 확인할 수 있다.
- 행렬곱 유도 섹션이 `[a b; c d][x; y]` 성분 계산을 단계적으로 보여준다.
- 모바일(세로)과 데스크톱(좌우 배치)에서 레이아웃이 깨지지 않는다.
- 빌드 과정 없이 정적 파일만으로 배포된다.

## 3. 수학 모델

변환 행렬:
```
M = | a  b |
    | c  d |
```

점의 변환 (행렬 × 벡터):
```
M · (x, y) = (a·x + b·y,  c·x + d·y)
```

기저벡터 관점 (핵심 직관):
```
i = (1, 0) → (a, c)   (M의 1열)
j = (0, 1) → (b, d)   (M의 2열)
```
→ 변환 후 도형은 "새 기저 `(a,c)`와 `(b,d)`가 만드는 격자"에 얹힌 모습이다.

행렬식 (넓이 배율):
```
det(M) = a·d − b·c
```
- `|det|` = 도형 넓이가 몇 배가 되는지. `det < 0` = 뒤집힘(대칭 포함). `det = 0` = 한 직선/점으로 붕괴.

프리셋 행렬:
- **회전(θ)**: `[[cosθ, −sinθ], [sinθ, cosθ]]`
- **확대/축소(s)**: `[[s, 0], [0, s]]`
- **전단(shear, k)**: `[[1, k], [0, 1]]`
- **x축 대칭**: `[[1, 0], [0, −1]]`
- **항등(초기화)**: `[[1, 0], [0, 1]]`

**교과 연결 포인트**
- 행렬 × 벡터 성분 계산 = 행렬의 곱셈 정의
- 두 변환의 합성 = 행렬의 곱 → **곱셈의 비교환성**(회전·전단 순서를 바꾸면 결과가 다름)을 시각화
- det의 부호·크기로 변환의 기하적 의미 해석

## 4. 화면 구성 (한 페이지 스크롤)

1. **인트로** — "AI는 왜 행렬로 세상을 볼까?" 문제 제기. 한 문장 동기 부여.
2. **시각화기** (핵심)
   - 격자 좌표평면(Canvas), 원점 중심, x·y축 표시
   - 원본: 단위정사각형(옅게) + 기저벡터 화살표 `i, j`
   - 변환 결과: 변형된 도형 + 변환된 기저벡터를 색으로 겹쳐 표시
   - 컨트롤: 슬라이더 a,b,c,d (범위 예: −2~2) + 프리셋 버튼(회전/확대/전단/대칭/초기화)
   - 패널: 현재 행렬, det 값(=넓이 배율)과 그 의미(뒤집힘/붕괴) 실시간 표시
3. **행렬곱 유도** — `[a b; c d][x; y] = (ax+by, cx+dy)` 성분 계산을 단계별 카드로. 기저 이동 관점 설명. 합성 순서에 따른 비교환성 언급.
4. **AI 탐구 결론** — 신경망 층(가중치행렬×입력벡터), 컴퓨터그래픽스의 좌표 변환, det의 의미, 곱셈 비교환성이 AI에서 갖는 함의. 모델의 한계·확장(3D·고차원 행렬).

## 5. 코드 구조 (역할 분리)

```
index.html          — 시맨틱 마크업, 4개 섹션
styles.css          — 반응형 스타일(모바일 우선), CSS 변수 테마
js/matrix.js        — 순수 수학: apply(M,v), determinant(M), presets(회전/확대/전단/대칭/항등), multiply(A,B) — DOM 무관, 단위 테스트 가능
js/matrix.test.mjs  — matrix.js 단위 테스트 (node --test)
js/canvas.js        — Canvas 렌더링: 좌표변환, 격자, 축, 도형, 기저벡터 화살표, 리사이즈 대응
js/app.js           — UI 연결: 슬라이더/프리셋 이벤트, 행렬·det 패널 갱신, 유도/결론 콘텐츠 주입
README.md           — 프로젝트 설명 + GitHub Pages 배포 방법
```

**경계 설계 원칙**
- `matrix.js`는 순수 함수만 — 입력(행렬·벡터)→출력(변환점·det·프리셋 행렬). DOM·Canvas를 모름 → 독립 검증 가능.
- `canvas.js`는 "어떻게 그릴지"만 담당. 그릴 데이터(행렬)는 외부에서 주입.
- `app.js`만 DOM과 상태를 조율. 각 파일은 단독으로 역할이 명확.

## 6. 반응형 / 모바일 최적화

- 모바일 우선 CSS, 미디어 쿼리로 데스크톱 확장.
- 데스크톱: 격자 + 컨트롤/패널 좌우 배치 / 모바일: 세로 스택.
- Canvas는 컨테이너 크기에 맞춰 리사이즈, `devicePixelRatio` 보정으로 고해상도 선명도 유지. 격자는 정사각 비율 유지(왜곡 방지).
- 슬라이더는 네이티브 `<input type="range">` → 터치 입력 자동 지원, 충분한 터치 타깃 크기.
- 폰트·여백은 `clamp()` 등으로 화면 크기에 따라 가변.
